 <summary><h3>👨‍💻 David's Coding Journey</h3></summary>

My path into software started with block-based coding in middle school, dragging logic blocks together before I'd written a real line of syntax. In high school I moved to the fundamentals: HTML and CSS to build my first web pages, then JavaScript to make them actually do something. That progression, from blocks to markup to a real language, is where I learned that I liked making things work, not just making them exist.

College is where the depth came. I picked up Java and object-oriented design, Python for AI and backend work, and SQL for working with data seriously. Through three engineering roles and a stack of projects, I went from writing scripts that ran on my own machine to architecting systems other people use: REST and GraphQL APIs, relational schemas, and the database and infrastructure layers underneath an application.

Today I work mostly in Go and PostgreSQL, building Resin, a knowledge graph platform with recursive graph traversal and JSONB-backed queries. The difference between now and the kid dragging code blocks isn't the number of languages I know, it's how I think: in systems, tradeoffs, and what should exist in the first place, not just how to make a single thing run.
